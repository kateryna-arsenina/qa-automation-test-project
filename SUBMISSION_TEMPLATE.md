## 1. What I built

A disciplined Python test suite for the reqres.in public REST API, covering three test layers (unit, integration, smoke), a CI workflow, and an HTML report.

## 2. Framework & SUT choice (done by hand)

| Decision | Choice | Rationale |
|---|---|---|
| Framework | pytest + requests | De-facto Python API testing standard; first-class HTML reporting; zero-friction GitHub Actions integration |
| SUT | reqres.in | Public REST sandbox mirroring realistic CRUD; no local service required |

## 3. Validator (written by hand)

`src/validators.py` — four validator functions, 5–10 lines each:
- `validate_user(data)` — checks required fields and types
- `validate_list_response(payload)` — checks pagination keys
- `validate_create_response(payload)` — checks id + createdAt
- `validate_register_payload(payload)` — rejects missing email/password

## 7. How to run

```bash
pip install -r requirements.txt
make test                       # full suite → reports/report_reqres_<timestamp>.html (auto-opens)
make demo                       # transcript header + full suite
make smoke                      # smoke only with report
pytest tests/unit -v            # unit only (no network, no report)
pytest tests/integration -v     # integration only
```
